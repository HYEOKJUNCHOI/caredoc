# START.md — 프로젝트 온보딩 스크립트

이 파일을 읽은 AI는 아래 순서를 **즉시** 실행한다.
사용자의 추가 지시 없이 Step 1~4를 순서대로 완료한 뒤 대기한다.

---

## 실행 순서

### Step 1. 규칙 파일 읽기

상위 폴더에서 AI에 맞는 파일을 읽는다:
- Claude Code → `CLAUDE.md`
- Antigravity / Gemini → `GEMINI.md`

> 협업 규칙, 코딩 기준, 레이아웃 프로토콜이 모두 담겨 있다. 반드시 읽는다.

---

### Step 2. 프로젝트 파악

현재 프로젝트 폴더에서 읽는다:
- `PROJECT.md` — 서비스 개요, 핵심 기능, 인터랙션 원칙, 타겟 유저

---

### Step 3. 이해 확인 응답

아래 형식으로 보고하고 대기한다:

```
✅ 온보딩 완료

프로젝트: [프로젝트명 — 한줄 요약]
스택: [주요 기술]
현재 상태: 신규 시작

다음 작업 후보:
1. [첫 번째 구현 항목]
2. [...]
3. [...]

무엇부터 시작할까요?
```

---

### Step 4. 첫 커밋

Step 3 보고 후 사용자 지시가 없어도 아래를 즉시 실행한다.

```
git add .
git commit -m "{프로젝트명}_스타트"
```

---

## 주의사항

- Step 4 완료 후 **사용자 지시 전까지 코드를 작성하지 않는다.**
- 읽다가 모호한 부분이 있으면 보고 직후 질문한다.
- `CLAUDE.md` / `GEMINI.md`의 모든 규칙은 이 세션 전반에 적용된다.
